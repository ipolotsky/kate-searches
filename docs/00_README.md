# KateSearches — пакет продуктовых и технических артефактов

> Рабочее название проекта: **KateSearches** (по имени первого клиента — маркетолога Kate из **LOOTON**).
> Цель сервиса: автоматизировать конвейер «новости отрасли → отбор релевантного → черновик SEO/AEO-поста, связывающий инфоповод с брендом клиента → дашборд черновиков».

Дата подготовки: 2026-06-30. Автор: техлид-сессия (ресёрч + упаковка).

---

## Что это за продукт (в одном абзаце)

B2B SaaS, который ежедневно мониторит источники новостей в нише клиента, **скорит каждую новость по кастомным критериям релевантности его аудитории**, и для прошедших отбор **генерит черновик статьи/поста в голосе бренда**, органично связывая инфоповод с брендом клиента в статью-симбиоз и упаковывая всё под SEO и AEO (индексация поисковиками и AI-движками). Маркетолог утром открывает дашборд и видит готовые черновики «откуда новость → что AI предложил → что прописать в SEO-разметке», правит и публикует. Сервис копит обратную связь и стоимость AI-запросов на каждого клиента.

Первый клиент — **LOOTON**, ресейл-маркетплейс премиум-одежды (streetwear / luxury / винтаж / архив / кроссовки). Её документы («Источники модных новостей», «Критерии отбора») фактически уже задают спеку движков отбора и экстракции — см. `05_ai_pipeline_prompts.md`.

## Зафиксированные решения (decision log)

| # | Развилка | Решение | Почему |
|---|----------|---------|--------|
| D1 | Стек | **Next.js (App Router) + Flowbite React** фронт, **FastAPI** AI/скрапинг-сервис | Чистая интеграция купленного Flowbite Pro + Python там, где живут либы скрапинга и LLM |
| D2 | Скоуп MVP | **Полноценный self-serve SaaS** с регистрацией/онбордингом с первого дня | Решение владельца; продукт «для всех», LOOTON — первый клиент-пилот |
| D3 | Биллинг | **В MVP биллинга нет, только cost-metering** | Kate ведём по договорённости; $-метрики копим сразу (нужны для тарифов и апселла). Stripe прикручиваем сразу после MVP |
| D4 | Юрисдикция | **Глобально / не-РФ** | Stripe + прямой доступ к OpenAI/Anthropic/Gemini без прокси |

> ⚠️ Напряжение D2 vs D3: «полный self-serve» обычно подразумевает биллинг. Разрешаем так: онбординг и мультитенант — с первого дня, тарификация — free-пилот / ручная, Stripe Billing подключаем первым же спринтом после MVP. Все экраны и схема данных проектируются с расчётом на биллинг (поля plan/usage есть с самого начала).

## Глоссарий

- **AEO** (Answer Engine Optimization) / **GEO** (Generative Engine Optimization) — оптимизация под цитирование в ответах AI-поисковиков (ChatGPT, Perplexity, Google AI Overviews).
- **Адаптер источника** — пер-источниковый модуль, приводящий любой источник (RSS, скрапер, соцсеть) к единой канонической `Document`-схеме.
- **Scoring / relevance score** — численная оценка релевантности новости аудитории конкретного клиента по его критериям.
- **Тенант** — клиент (компания). Данные строго изолированы по тенанту.
- **COGS** — себестоимость (для нас в основном = стоимость AI-запросов).
- **Cost-metering** — учёт стоимости каждого AI-запроса в разрезе тенанта/пользователя/стадии.
